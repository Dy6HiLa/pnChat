# pnChat

pnChat - плагин для Paper-сервера Minecraft, который заменяет стандартный чат на систему с локальным, глобальным и административным каналом. Он нужен, чтобы игроки не засоряли весь сервер обычными разговорами, могли быстро написать всем через глобальный чат, а администрация имела отдельный рабочий канал.

## Как работает чат

По умолчанию любое сообщение игрока отправляется в локальный чат. Это значит, что его увидят только игроки в том же мире и только в радиусе, указанном в `config.yml`.

Например, если радиус стоит `100`, сообщение увидят игроки на расстоянии до 100 блоков от отправителя. Игроки в другом мире или дальше этого расстояния сообщение не получат. Если рядом никого нет, плагин сообщает отправителю, что сообщение доставлено только ему.

Глобальный чат включается символом `!` в начале сообщения. Игрок пишет сообщение с `!`, плагин убирает этот символ и отправляет текст всем онлайн-игрокам сервера.

Админ-чат работает как отдельный режим. Администратор включает его командой `/pnchat ac`, после этого его обычные сообщения начинают уходить только другим игрокам с правом `pnchat.admin`. Повторная команда выключает этот режим.

## Упоминания игроков

pnChat умеет находить ник онлайн-игрока в сообщении и превращать его в упоминание. Работают два варианта:

- игрок пишет ник через `@`;
- игрок пишет обычный ник без `@`.

Если такой игрок сейчас онлайн, плагин подставляет формат из настройки `mention-format` и проигрывает ему звук уведомления. Ник сравнивается без учёта регистра, поэтому `Steve`, `steve` и `STEVE` будут найдены как один и тот же онлайн-игрок.

Автоупоминания без `@` можно настроить:

```yaml
plain-name-mentions-enabled: true
plain-name-mention-min-length: 3
plain-name-mention-ignored-words: []
```

`plain-name-mentions-enabled` включает или выключает поиск ников без `@`.

`plain-name-mention-min-length` задаёт минимальную длину ника, который можно превратить в упоминание. Это защищает от случайных совпадений с короткими словами.

`plain-name-mention-ignored-words` нужен для исключений. Если на сервере есть ник, который часто совпадает с обычным словом, добавьте его в этот список.

## Модерация сообщений

Перед отправкой сообщения pnChat проверяет текст. Если сообщение не проходит проверку, оно не отправляется другим игрокам, а отправитель получает понятное сообщение из `messages.yml`.

Плагин проверяет:

- слишком частые сообщения;
- злоупотребление CAPS LOCK;
- слишком длинный текст;
- большое количество одинаковых символов подряд;
- запрещённые слова из `blocked-words.txt`;
- ссылки, если включена фильтрация ссылок.

Ссылки проверяются отдельно. Если `link-moderation-enabled` включён, плагин ищет URL, Discord-инвайты и Telegram-ссылки. Разрешённые домены указываются в `allowed-links.txt`. Всё, чего нет в этом списке, блокируется.

## Интеграции

pnChat не требует дополнительных плагинов для основной работы, но умеет использовать их, если они установлены.

LuckPerms используется для префикса игрока в чате. Если LuckPerms недоступен, чат продолжит работать без префикса.

PlaceholderAPI используется для обработки плейсхолдеров в сообщениях. Если PlaceholderAPI не установлен, текст отправляется без подстановки плейсхолдеров.

Essentials, LiteBans, AdvancedBan и CMI используются для проверки mute-статуса. Если игрок замьючен одним из этих плагинов, pnChat не даст ему писать в чат.

## Команды

`/pnchat reload` перезагружает `config.yml`, `messages.yml`, `allowed-links.txt`, `blocked-words.txt` и настройки внутренних сервисов без перезапуска сервера.

`/pnchat ac` включает или выключает режим админ-чата для игрока.

`/pnchat adminchat` делает то же самое, что `/pnchat ac`.

## Права

`pnchat.admin` даёт доступ к командам pnChat, админ-чату и уведомлениям об обновлениях. По умолчанию право есть у операторов сервера.

`pnchat.color` разрешает использовать цветовые коды, если в конфиге включено требование права на цветной чат.

Также плагин учитывает `pnchat.format` и `pnchat.chatcolor` как дополнительные права на форматирование сообщений.

## Настройка

После первого запуска плагин создаёт папку:

```text
plugins/pnChat
```

Главные файлы:

```text
config.yml
messages.yml
allowed-links.txt
blocked-words.txt
```

В `config.yml` настраивается поведение чата: радиус локального канала, форматы сообщений, звук упоминаний, права на цвета, ограничения сообщений, антиспам и фильтрация ссылок.

В `messages.yml` лежат сообщения, которые видят игроки при ошибках, блокировках и перезагрузке конфига.

В `allowed-links.txt` указываются домены и части ссылок, которые разрешено отправлять в чат.

В `blocked-words.txt` указываются слова и фразы, которые должны блокироваться.

После изменения настроек используйте:

```text
/pnchat reload
```

## Форматы сообщений

Формат локального, глобального и админ-чата задаётся в `config.yml`.

```yaml
local-format: '&8[&aL&8] &7{player} &8» &f{message}'
global-format: '&8[&dG&8] &f{player} &8» &f{message}'
admin-format: '&8[&cADMIN&8] &f{player}&7: &f{message}'
```

`{player}` заменяется на ник отправителя.

`{message}` заменяется на обработанный текст сообщения.

Цвета указываются через стандартные Minecraft-коды с `&`. Hex-цвета можно включить или выключить настройкой `allow-hex-colors`.

## Установка

1. Скачайте `pnChat-1.0.0.jar` из GitHub Releases.
2. Положите файл в папку `plugins` вашего Paper-сервера.
3. Перезапустите сервер.
4. Настройте файлы в `plugins/pnChat`.
5. Используйте `/pnchat reload`, если меняли настройки после запуска.

## Сборка из исходников

Для сборки нужна Java 21.

```powershell
.\gradlew.bat build
```

Готовый файл появится здесь:

```text
build/libs/pnChat-1.0.0.jar
```
