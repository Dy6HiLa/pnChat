# pnChat

pnChat - плагин чата для Paper-серверов Minecraft. Он разделяет сообщения на
локальный, глобальный и административный чат, форматирует сообщения и применяет
базовую модерацию перед отправкой игрокам.

## Возможности

- Локальный чат в радиусе, заданном в `config.yml`.
- Глобальный чат через префикс `!` перед сообщением.
- Админ-чат с переключением через `/pnchat ac` или `/pnchat adminchat`.
- Настраиваемые форматы локального, глобального и административного чата.
- Упоминания игроков через `@nickname` с подсветкой и звуковым уведомлением.
- Поддержка цветовых кодов `&` и hex-цветов.
- Фильтрация спама, CAPS, слишком длинных сообщений, повторяющихся символов,
  ссылок и запрещённых слов.
- Белый список разрешённых доменов в `allowed-links.txt`.
- Список запрещённых слов в `blocked-words.txt`.
- Мягкая интеграция с LuckPerms и PlaceholderAPI, если они установлены.
- Проверка mute-статуса через Essentials, LiteBans, AdvancedBan и CMI, если эти
  плагины установлены на сервере.
- Автоматическая проверка обновлений и уведомление администраторов.

## Требования

- Java 21.
- Paper API `1.21.4`.
- Gradle Wrapper из репозитория.

Опциональные плагины:

- LuckPerms - префиксы игроков.
- PlaceholderAPI - обработка плейсхолдеров в сообщениях.
- Essentials, LiteBans, AdvancedBan или CMI - проверка mute-статуса.

## Сборка

```powershell
.\gradlew.bat build
```

Готовый JAR будет создан в:

```text
build/libs/pnChat-1.0.0.jar
```

## Установка

1. Соберите проект или скачайте готовый JAR из релиза.
2. Поместите JAR в папку `plugins` Paper-сервера.
3. Запустите или перезапустите сервер.
4. После первого запуска настройте файлы в `plugins/pnChat`.
5. Примените изменения командой `/pnchat reload` или перезапустите сервер.

## Использование чата

Обычное сообщение отправляется в локальный чат. Получатели выбираются среди
игроков в том же мире в радиусе `radius`.

```text
Привет рядом
```

Сообщение с `!` в начале отправляется всем онлайн-игрокам:

```text
!Привет всем
```

После включения админ-чата все обычные сообщения администратора отправляются
только игрокам с правом администратора:

```text
/pnchat ac
```

## Команды

| Команда | Описание | Право |
| --- | --- | --- |
| `/pnchat reload` | Перезагружает конфигурацию, сообщения и списки модерации. | `pnchat.admin` |
| `/pnchat ac` | Включает или выключает режим админ-чата для игрока. | `pnchat.admin` |
| `/pnchat adminchat` | Алиас команды `/pnchat ac`. | `pnchat.admin` |

## Права

| Право | Назначение | По умолчанию |
| --- | --- | --- |
| `pnchat.admin` | Доступ к `/pnchat`, админ-чату и уведомлениям об обновлениях. | `op` |
| `pnchat.color` | Разрешение на цветовые коды, если включён `color-permission-required`. | не задано |
| `pnchat.format` | Дополнительное право на форматирование сообщений. | не задано |
| `pnchat.chatcolor` | Дополнительное право на цветной чат. | не задано |

## Конфигурация

Основные параметры находятся в `config.yml`.

| Параметр | Описание |
| --- | --- |
| `radius` | Радиус локального чата в блоках. |
| `local-format` | Формат локального чата. |
| `global-format` | Формат глобального чата. |
| `admin-format` | Формат админ-чата. |
| `mention-format` | Формат подсветки упоминаний. |
| `mention-sound-enabled` | Включает звук при упоминании игрока. |
| `mention-sound` | Bukkit-ключ звука для упоминаний. |
| `mention-sound-cooldown-ms` | Кулдаун звука упоминаний для одного игрока. |
| `admin-chat-enabled` | Конфигурационный флаг админ-чата. |
| `admin-chat-permission` | Право доступа к админ-чату. |
| `color-permission-required` | Требовать право для цветовых кодов. |
| `color-permission` | Право на использование цветовых кодов. |
| `allow-hex-colors` | Разрешает hex-цвета. |
| `max-message-length` | Максимальная длина сообщения. |
| `link-moderation-enabled` | Включает фильтрацию ссылок. |
| `spam-cooldown-ms` | Интервал для антиспам-проверки. |
| `spam-max-messages` | Допустимое количество сообщений за интервал. |
| `caps-threshold` | Доля заглавных букв, после которой сообщение блокируется. |
| `max-repeated-chars` | Максимальное количество повторяющихся символов подряд. |

Сообщения игрокам настраиваются в `messages.yml`.

## Списки модерации

- `allowed-links.txt` - разрешённые домены или части URL. Если сообщение
  содержит ссылку, она должна совпасть с одним из разрешённых значений.
- `blocked-words.txt` - запрещённые слова и фразы. Проверка выполняется после
  нормализации текста.

После изменения списков выполните:

```text
/pnchat reload
```

## Плейсхолдеры форматов

В форматах чата используются:

- `{player}` - имя отправителя.
- `{message}` - обработанный текст сообщения.

Пример:

```yaml
local-format: '&8[&aL&8] &7{player} &8» &f{message}'
global-format: '&8[&dG&8] &f{player} &8» &f{message}'
admin-format: '&8[&cADMIN&8] &f{player}&7: &f{message}'
```
